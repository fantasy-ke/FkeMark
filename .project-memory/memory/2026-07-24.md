## 2. 长文档实时编辑停止自动整篇序列化

### 需求
- 超长 Markdown 在实时编辑视图输入后仍会反复出现长时间无响应；阅读、分栏和源码视图不受影响。

### 实现
- 保留 100000 字符的长文档阈值；长文档输入时只记录待处理编辑器并立即标记 dirty，不再在停止输入 300ms 后自动执行整篇 `getHTML()` 与 `htmlToMarkdown()`。
- 保存、自动保存、导出、切换模式及标签页等现有 `EditorHandle.getContent()` 边界仍会强制刷新最新内容，避免输入丢失。
- 短文档继续即时同步父级内容，行为不变。

### 涉及文件或模块
- `src/components/editor/useDeferredEditorChange.ts`
- `tests/editor.performance.test.tsx`
- `.project-memory/memory/MEMORY.md`

### 验证结果
- `npm test -- --run tests/editor.performance.test.tsx`：通过，4 个测试。
- `npm run build`：通过，1949 个模块完成构建。
- `git diff --check`：通过，仅有 Git 的 Windows 换行提示。
- 回归测试覆盖连续输入后停顿 10 秒仍不触发整篇序列化，以及 `getContent()` 会同步最新待保存输入。
- 本次修改的代码文件分别为 75 行和 155 行，均未超过 800 行限制。

### 已知限制
