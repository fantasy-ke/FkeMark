## 第六轮：Tauri 无边框窗口圆角不生效修复

### 问题诊断
用户反馈：软件主页面与关闭/最小化/最大化窗口按钮的圆角没生效，怀疑是 tauri 壳问题。

根因定位：`tauri.conf.json` 已设 `decorations: false`（无边框自绘窗口），但 `transparent: false`（不透明窗口）。在 Tauri 中，**只有 `transparent: true` 才能让 CSS 的 `border-radius` 真正"挖掉"窗口矩形的角落**——否则底层窗口仍是矩形，CSS 圆角只裁剪了应用内容但矩形角落仍可见。

### 修改清单
1. **`src-tauri/tauri.conf.json`**：`"transparent": false` → `"transparent": true`
2. **`src/index.css`**：
   - `body { background: transparent; }`（原为 `var(--bg)`，让窗口底层透明）
   - `.app-container` 加 `border-radius: var(--radius-base); overflow: hidden; border: 1px solid var(--border); box-shadow: 0 8px 40px rgba(0,0,0,0.18), 0 2px 8px rgba(0,0,0,0.1); transition: border-radius 200ms ease;`（圆角 + 边框 + 阴影）
   - `body.maximized .app-container { border-radius: 0; border: none; box-shadow: none; }`（最大化时移除圆角与边框，避免裁切屏幕边缘）
   - `.win-btn { border-radius: var(--radius-btn); }`（原 `border-radius: 0`，窗口按钮圆角生效）

## 第八轮：修复首次打开 md 文件显示无格式文本的 bug

### 问题现象
点开 md 文件，第一次显示文本都是没换行没格式的展示（原始 markdown 纯文本），再次点击时格式样式才恢复。

### 根因分析（初始化时序问题）
- `src/App.tsx` line 461：`showWelcome = !currentFile && !fileContent`。首次打开应用时 Editor 不挂载（显示 WelcomeScreen）。
- 用户点击文件打开 → `setFileContent(原始markdown)` + `setCurrentFile(path)` → `showWelcome` 变 false → **Editor 首次挂载**，此时 `fileContent` 已是原始 markdown 文本。
- `src/components/Editor.tsx` line 98（原）：`content: content || '<p></p>'` —— useEditor 初始化时直接把**原始 markdown 文本**塞给 TipTap。TipTap 把它当 HTML 解析，但 markdown 非合法 HTML → 渲染为无格式的纯文本块。
- line 138-143 的 useEffect 同步逻辑虽然在首次挂载后会触发 `setContent(markdownToHtml(content))` 修正，但 useEffect 在 paint 之后异步执行，导致用户先看到无格式文本，修正后才恢复格式（闪烁）。

### 修复
`src/components/Editor.tsx` line 98：
```diff
- content: content || '<p></p>',
+ // 初始化时即把 markdown 转为 HTML，避免首次渲染显示无格式的原始文本
+ content: markdownToHtml(content || ''),
```
`markdownToHtml` 是 function 声明（hoisted），在 useEditor 调用位置可用。TipTap 初始化时即拿到正确 HTML，首次渲染即正确格式，无需等待 useEffect 修正。

- tsc 0 错误、vite build 通过（CSS 41.65KB / JS 723.83KB gzip 227KB）。tauri build 后台打包中（任务 Mg4RlZ）。
