## 修复4：全文检索输入时焦点被抢到文本域（commit `2c1606a`）

### 问题
源码/分栏模式下在检索框中输入——每打一个字符焦点就跳到文本域，导致完全无法连续输入；Enter 键也会触发 findNext/findPrev 并抢焦点。

### 根因
`FindReplaceBar.selectTextMatch()` 调用 `ta.focus()` 强制把焦点从检索框拉回 textarea：
- 每输入字符 → `setFindText` → `doSearch()` → `selectTextMatch(0)` → `textarea.focus()` → 焦点跳走
- 按 Enter → `handleKeyDown` → `findNext()` → `updateCurrentMatch()` → `selectTextMatch()` → `textarea.focus()` → 焦点跳走

### 修复
移除 `selectTextMatch` 中的 `ta.focus()`，保留 `setSelectionRange` + `scrollTop`：
- `setSelectionRange` 在非聚焦 textarea 上可安全调用（内部设置选区，重新获焦时可见）
- `scrollTop` 无需焦点即可滚动到匹配位置
- 检索框焦点保持，用户可连续输入；Enter 键 findNext 只滚动不抢焦点
- 编译/构建/测试均通过（56/57）

## 修复5：图片文件名含下划线被 turndown 转义为 \_ 导致显示异常（commit `23443e8`）

### 问题
拖入/上传 `fantasyke_Arc.png` 后，Markdown 源码中 alt 文本变成 `[fantasyke\_Arc]`（多了反斜杠），
图片在实时编辑和预览视图中显示"加载失败"。用户手动删除 `\` 后恢复正常。

### 根因
turndown 默认 image 规则会对 alt 文本调用 `escape()`，将 `_` 转为 `\_`。
虽然 `\_` 在 Markdown 中理论合法（渲染为字面下划线），但通过 `markdownToHtml` →
`setContent` 回灌 ProseMirror 后，反斜杠可能污染图片节点属性，进而导致渲染/加载异常。

具体的污染路径：`![fantasyke\_Arc.png](path)` → 内置引擎 regex 解析 →
`<img alt="fantasyke\_Arc.png" src="...">` → TipTap `setContent` 解析 HTML →
ProseMirror image node `attrs.alt = "fantasyke\_Arc.png"`（反斜杠滞留节点属性中），
后续渲染可能因属性中的转义字符与 ProseMirror 内部处理冲突。

### 修复
`markdown.third.ts` 的 `createTurndown()` 中新增 `imagePlain` 规则：
- filter 匹配所有 `<img>`（无 width/height 样式），已由 `imageWithSize` 规则优先处理有尺寸图片
- replacement 直接使用 `node.getAttribute('alt')` 原始值，**不调用 escape()**
- Markdown 图片 alt 文本中的 `_` 不会被解析为强调语法（CommonMark 规范：`![]() `中 alt 为字面文本），无需转义

## 功能：源码/分栏模式检索增加多匹配高亮（commit `8a257cb`）

### 需求
源码和分栏视图的检索仅靠 textarea `setSelectionRange` 选中当前匹配，缺乏类似浏览器 Ctrl+F 的多匹配高亮效果。

### 实现方案
在 textarea 之上叠加 `pointer-events:none` 覆盖层，mirror 相同字体/行高/内边距，用 `<mark>` 渲染所有匹配位置。

### 架构
- **SearchHighlightOverlay 组件**（新文件 `src/components/editor/SearchHighlightOverlay.tsx`）：
  - 接受 `text/matches/currentIndex/scrollTop/isSplit` props
  - `splitSegments()` 按匹配位置将文本拆分为高亮/非高亮片段数组
  - 通过 `transform: translateY(-scrollTop)` 同步 textarea 滚动（`overflow:hidden` + transform，非滚动容器）
  - 无匹配时返回 null（不渲染 DOM）
- **FindReplaceBar** 新增 `onTextMatchesChange` 回调 prop：
  - `doSearch` / `updateCurrentMatch` 文本模式分支调用该回调传出 `(matches[], currentIndex)`
  - 查找栏关闭/findText 清空时传出 `([], -1)` 清除高亮
- **Editor.tsx** 集成：
  - 新增 `searchMatches` / `searchCurrentIdx` / `textareaScrollTop` 状态
  - 源码/分栏 textarea 外层包裹 `position:relative` 容器（`.source-textarea-wrapper`）
  - `onScroll` 捕获 `scrollTop` 驱动覆盖层同步
  - `FindReplaceBar` 传入 `onTextMatchesChange` 更新状态
  - 分栏模式下传 `isSplit` 标识（覆盖层不应用居中 max-width）

### 样式
- `.search-highlight-overlay`：覆盖 textarea 全区域，`pointer-events:none` 穿透点击/滚动
- `.search-highlight-mark`：`background: rgba(var(--accent), 0.25)` 浅色标记
- `.search-highlight-mark--current`：`background: rgba(var(--accent), 0.55)` + `outline: 1px solid var(--accent)` 突出当前
- `.search-highlight-plain`：`color: transparent`（非匹配文字透出底层 textarea 的文字颜色）
- `.source-textarea-wrapper`：`position:relative; flex:1; overflow:hidden` 作为定位锚点
