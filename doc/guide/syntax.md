# Markdown 语法

FkeMark 支持常见 Markdown 写作语法，并在 Live / Read 模式中把大部分结构渲染成更接近成稿的效果。

## 基础语法速查

| 类型 | 写法示例 | 用途 |
| --- | --- | --- |
| 标题 | <code># 一级标题</code> | 组织文章层级 |
| 加粗 | <code>**重点**</code> | 强调关键词 |
| 斜体 | <code>*术语*</code> | 标记语气或术语 |
| 删除线 | <code>~~旧内容~~</code> | 标记已废弃内容 |
| 引用 | <code>&gt; 引用内容</code> | 摘录、提示或说明 |
| 无序列表 | <code>- 条目</code> | 普通列表 |
| 有序列表 | <code>1. 步骤</code> | 流程步骤 |
| 任务列表 | <code>- [x] 已完成</code> | 清单和待办 |
| 链接 | <code>[标题](https://example.com)</code> | 引用外部或内部资料 |
| 图片 | <code>&#33;[说明](./assets/a.png)</code> | 插入截图或配图 |
| 行内代码 | <code>const name = 'FkeMark'</code> | 标记变量、命令或短代码 |

## 代码块

使用三个波浪线或三个反引号包住代码，并写上语言名称：

~~~~md
~~~ts
const mode = 'live'
console.log(mode)
~~~
~~~~

在编辑器中输入 <code>&#96;&#96;&#96;ts</code> 后回车，也可以快速生成代码块。

## 表格

~~~md
| 模式 | 适合场景 |
| --- | --- |
| Live | 日常写作 |
| Read | 阅读校对 |
| Source | 精确编辑 |
~~~

表格内容较宽时，优先拆短列或拆成多个小表格，阅读体验会更稳定。

## Front Matter 和标签

可以在文档开头写 YAML Front Matter，用于记录标题、日期、分类等元信息：

~~~md
---
title: 项目记录
date: 2026-07-23
tags:
  - markdown
  - fkemark
---
~~~

正文中也可以使用 <code>#tag</code> 形式保留轻量标签。

## 脚注

~~~md
这里需要补充说明。[^note]

[^note]: 这是一条脚注内容。
~~~

## 数学公式

行内公式适合短表达式，块级公式适合独立展示：

~~~md
行内公式：$a^2 + b^2 = c^2$

$$
E = mc^2
$$
~~~

## 写作建议

- 一篇文档建议只保留一个一级标题。
- 链接文字要有意义，不要只写“点击这里”。
- 代码块加语言名称，便于高亮和后续检索。
- 图片建议写清楚替代文本，方便阅读和归档。
