# 编辑教程

## 选择编辑模式

FkeMark 不强迫你只在源码或预览之间二选一。

- **Live**：适合日常写作。常见 Markdown 块会即时渲染，减少排版干扰。
- **Read**：适合校对和阅读。把文档当作成稿来检查。
- **Source**：适合精确修改。保留完整 Markdown 源码控制权。

## 常用块级内容

你可以在文档里使用：

- 标题、段落、引用
- 有序 / 无序列表、任务列表
- 表格
- 代码块与语法高亮
- KaTeX 数学公式
- 图片、链接、高亮、下划线

## 推荐写作方式

1. 用 `Source` 快速起草结构。
2. 切到 `Live` 补充正文和表格。
3. 开启分屏核对复杂代码块或公式。
4. 用 `Read` 做最终阅读检查。

## 导入与导出

FkeMark 支持 Markdown、HTML、TXT 之间转换。建议把 Markdown 作为主格式保存，导出格式只用于分享或迁移。

## 写长文的小建议

- 每篇文章只保留一个一级标题。
- 大段内容拆成二级 / 三级标题。
- 表格过宽时优先拆成短表格，少依赖横向滚动。
- 重要链接使用明确文字，不要只写“点击这里”。
