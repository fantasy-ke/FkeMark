## 第十轮：小地图悬浮预览优化

### 需求
1. 小地图在左边时，悬浮预览框显示在右边；小地图在右边时，悬浮显示在左边。
2. 悬浮框样式调整为带小尖尖指向地图位置（类似消息气泡），内容区域渲染 markdown。
3. 悬浮预览根据视图模式显示内容：live/read 模式显示渲染后的 HTML，source 模式显示源码。

### 实现
**`src/components/Editor.tsx`** Minimap 组件重写：
- 新增 `side: 'left'|'right'` 和 `editorMode` props。
- `<Minimap>` 调用处传入 `side={minimapOnLeft ? 'left' : 'right'}` 和 `editorMode`。
- 悬浮预览改为提取以鼠标对应行为中心的 5 行片段：
  - `editorMode === 'source'`：用 escapeHtml 转义后包 `<pre class="minimap-tip-pre">` 显示源码。
  - `live`/`read` 模式：调 `markdownToHtml(fragment)` 渲染为 HTML，用 `dangerouslySetInnerHTML` 注入。
- tooltip 只存 `{ html, y }`，水平位置由 CSS class 决定（不再存 x 坐标）。
- 新增 `.minimap-tooltip-arrow` 元素作为小尖尖。
- 复用文件末尾已有的 `escapeHtml` 函数（删除重复定义）。

**`src/index.css`** tooltip 样式重写：
- `.minimap-tooltip` 改为固定 width 280px + `transform: translateY(-50%)` 垂直居中。
- `.minimap-tooltip-left` → `left: calc(100% + 14px)`（显示在右边）。
- `.minimap-tooltip-right` → `right: calc(100% + 14px)`（显示在左边）。
- `.minimap-tooltip-arrow` 三角形 + `::after` 内部白色覆盖留 1px 边框，左/右方向分别处理。
- `.minimap-tooltip-content` 完整 markdown 元素样式（h1-h4/ul/ol/code/pre/blockquote/table/hr/a/img）。
- `.minimap-tip-pre` 源码模式专用样式。
- 动画 keyframes 改为 `translateY(-50%)` 保持居中。

## 第十轮修复：悬浮窗不显示（overflow:hidden 裁切问题）

### 问题
用户反馈"没有看见悬浮窗显示出来"。

### 根因
`.minimap-panel` 有 `overflow: hidden`（为了让长小地图内容不溢出 panel），但 tooltip 是 panel 的子元素，`position: absolute` 定位到 panel 外部时会被 hidden 裁切掉，导致不可见。

### 修复（React Portal 方案）
**`src/components/Editor.tsx`**：
- 新增 `import { createPortal } from 'react-dom'`（注意：createPortal 不在 react 命名导出，必须从 react-dom 导入）。
- Minimap hover state 改为 `{ html, y, left }`（left 改为视口绝对坐标）。
- handleMouseMove 计算 tooltip 的视口坐标：`side==='left'` 时 `left = rect.right + 14`，`side==='right'` 时 `left = rect.left - 14 - 280`。
- tooltip 渲染改用 `createPortal(<div>...</div>, document.body)`，脱离 panel 的 overflow 限制。
- tooltip 的 `top`/`left` 直接用视口坐标（配合 fixed 定位）。

**`src/index.css`**：
- `.minimap-tooltip` 从 `position: absolute` 改为 `position: fixed`，`z-index` 1000。
- 删除 `.minimap-tooltip-left { left: calc(100% + 14px) }` / `.minimap-tooltip-right { right: calc(100% + 14px) }`（位置改由 JS 设定）。
- 保留 `.minimap-tooltip-arrow` 左右方向的小尖尖 border 规则。
- 清理重复的 `.minimap-tooltip-left/right .minimap-tooltip-arrow` 定义。

### 关键经验
- 父元素 `overflow: hidden` 会裁切 `position: absolute` 的子元素。解决方案：用 React Portal 把 tooltip 渲染到 `document.body`，配合 `position: fixed` + 视口坐标，彻底脱离父元素 overflow 限制。
- `createPortal` 必须从 `react-dom` 导入，不是 `react` 的命名导出（TS 报 TS2724）。

### 构建验证（第十轮修复）
- tsc 0 错误、vite build 通过（CSS 46.41KB / JS 728.90KB gzip 228KB）。tauri build 后台打包中（任务 BL1bIU）。

## 第十轮修复 2：表格右键菜单不生效

### 问题
用户反馈"表格右键菜单没有效果"。

### 根因（双重问题）
1. **close useEffect 监听了 contextmenu 事件**：用户右键表格时，contextmenu 事件冒泡到 document → close 监听器立即 `setTableCtxMenu(null)` → 菜单刚弹出就被关闭，用户根本看不到。
2. **菜单容器未阻止 contextmenu 冒泡**：在菜单内右键会再次触发 contextmenu → 关闭菜单。

### 修复
1. `onScrollContextMenu` 加 `e.nativeEvent.stopImmediatePropagation()` 阻止 contextmenu 事件冒泡到 document。
2. close useEffect **移除 contextmenu 监听**（只保留 click 关闭）。
3. 表格菜单容器加 `onContextMenu={(e) => { e.preventDefault(); e.stopPropagation() }}` 防止菜单内右键触发关闭。

### 构建验证
- tsc 0 错误、vite build 通过（CSS 46.41KB / JS 728.91KB gzip 228KB）。tauri build 后台打包中（任务 NIoqy7）。

## 第十二轮：3 项遗留问题修复

### 1. 右键菜单屏幕右下角溢出（无边界钳制）
**`src/components/Editor.tsx`**：
- 新增 `clampMenuPos(x, y, estW, estH)` 工具函数：水平 `pad ~ innerWidth-estW-pad`，垂直 `pad ~ innerHeight-estH-pad`，Math.min/max 钳制。
- `onScrollContextMenu` 中 `setTableCtxMenu` 与 `setContextMenuPos` 均经 `clampMenuPos` 处理（表格菜单 estH=300，通用菜单 estH=180）。

### 2. TopBar 切换主题忽略 system 选项
**`src/App.tsx`**：
- `handleToggleTheme` 从 `isDark ? 'light' : 'dark'`（仅二态）改为三态循环：`light → dark → system → light`。
**`src/components/TopBar.tsx`**：
- props `isDark: boolean` → `theme: 'light'|'dark'|'system'`，菜单项图标按 theme 显示太阳/月亮/电脑，右侧 `menu-shortcut` 显示当前模式（明亮/黑暗/系统）。
- App.tsx 渲染处 `isDark={isDark}` → `theme={settings.theme}`。

### 3. 新建文件时编辑器内容区域宽度不按设置配置
**根因**：`applySettings` 把 maxWidth 直接设在 `.ProseMirror` 上，但父容器 `.editor-inner`（及 `.source-textarea`）用 `max-width: var(--editor-max-w)`（默认 800px）截断，导致 editorWidth 设置（680/800/960）始终被 800px 上限卡住、不生效；新建文件（从欢迎页挂载）时尤为明显。
**修复**：`applySettings` 不再设 `.ProseMirror` 内联 maxWidth，改为 `document.documentElement.style.setProperty('--editor-max-w', ewMap[editorWidth])`，父容器 `.editor-inner` 与 `.source-textarea` 均引用该变量，宽度设置对所有文件（含新建）生效。
