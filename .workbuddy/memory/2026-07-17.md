## 第二轮 CI 报错修复（Windows/macOS rename + Ubuntu libsoup）
- Windows/macOS `failed to rename fke-mark(.exe) → FkeMark(.exe)` (os error 2, 源不存在)：根因 Cargo.toml 的 `[[bin]] name="FkeMark"` 让 cargo 直接产出 FkeMark(.exe)，但 Tauri 按 package 名 `fke-mark` 找源重命名 → 源缺失。修复：移除 `[[bin]]` 块，cargo 产出 `fke-mark(.exe)`，由 Tauri 正常重命名为 `FkeMark(.exe)`（产物名与 productName 对齐）。TOML 校验通过。
- Ubuntu `Package libsoup-2.4 was not found` / soup2-sys 失败：根因装的 `libwebkit2gtk-4.1-dev` 提供 libsoup3，而 Tauri v1 的 webkit2gtk 链接 libsoup-2.4。修复：改装 `libwebkit2gtk-4.0-dev` + 显式 `libsoup2.4-dev`。
- macOS `block v0.1.6` 仅为 future-incompat 警告，非失败，随 rename 修复一并解决。

## 第五轮 CI 结构重构（拆分 release/dev 工作流 + changelog 独立文件）
- 删除单一 `.github/workflows/build.yml`，拆分为两个独立工作流：
  - `.github/workflows/release.yml`：触发 `tags: [v*]`，正式 Release（draft 手动发布）
  - `.github/workflows/dev.yml`：触发 `branches: [dev]`，开发预发布版（tag `dev-latest`、name `Dev Build (dev-<短SHA7>)`、prerelease、自动发布），每次推送先 `gh release delete` + `gh api DELETE /git/refs/tags/dev-latest` 清理旧版再重建
- changelog 分类配置抽到独立文件 `.github/changelog.yml`（`categories:` 五类：新功能/问题修复/文档与流程/重构与优化/其他更新），两个工作流通过 `Load changelog config` 步骤（`echo "CHANGELOG<<EOF" >> $GITHUB_ENV; cat .github/changelog.yml >> $GITHUB_ENV; echo EOF`）读入 env，再 `changelog: ${{ env.CHANGELOG }}` 引用。
- 注意：GitHub Actions 要求工作流文件必须直接在 `.github/workflows/` 下，不支持子文件夹；故 changelog 配置放在 `.github/changelog.yml`（非 workflows 内），避免被当作无效工作流。
- 校验：pyyaml `safe_load` 三个文件全部通过；release/dev 触发条件、Load 步骤、tauri.changelog 引用均正确；changelog.yml 五分类解析正确。
