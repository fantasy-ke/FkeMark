## 第二轮 CI 报错修复（Windows/macOS rename + Ubuntu libsoup）
- Windows/macOS `failed to rename fke-mark(.exe) → FkeMark(.exe)` (os error 2, 源不存在)：根因 Cargo.toml 的 `[[bin]] name="FkeMark"` 让 cargo 直接产出 FkeMark(.exe)，但 Tauri 按 package 名 `fke-mark` 找源重命名 → 源缺失。修复：移除 `[[bin]]` 块，cargo 产出 `fke-mark(.exe)`，由 Tauri 正常重命名为 `FkeMark(.exe)`（产物名与 productName 对齐）。TOML 校验通过。
- Ubuntu `Package libsoup-2.4 was not found` / soup2-sys 失败：根因装的 `libwebkit2gtk-4.1-dev` 提供 libsoup3，而 Tauri v1 的 webkit2gtk 链接 libsoup-2.4。修复：改装 `libwebkit2gtk-4.0-dev` + 显式 `libsoup2.4-dev`。

### 技术决策说明
- **不使用 Tauri 内置 updater**：内置 updater 需要签名密钥对 + 签名每个构建 + `updater.active: true` + pubkey 配置。用户需求是"软件包在 GitHub 下载更新"，更适合自定义检查器 + 引导用户到 GitHub 下载

- **GitHub API CORS**：GitHub API 返回 `Access-Control-Allow-Origin: *`，Tauri WebView2 的 `fetch` 可直接调用，无需 Tauri HTTP allowlist

### 问题 1：Dev 构建版本号显示为正式版本号
- **根因**：CI dev 工作流未在构建前修改 Cargo.toml / package.json / tauri.conf.json 中的版本号，`getVersion()` 始终返回 "0.1.0"

### 问题 2：检查更新后始终给出提示（不管有无新版本）
- **根因**：`doCheckUpdate` 仅在 `info.isNewer && showLoading` 时显示 Toast，未覆盖"已最新"和"检查失败"场景
- **修复**：

## 修复：CI 版本号注入问题（三连修复）

### 问题 1：dev.yml `DEV_VERSION` 为空 + bash 语法错误
- **根因 A**：`dev-release` job 的 `outputs` 没有声明 `dev_version`，`build` job 拿不到值
- **根因 B**：`sed` 命令用双引号包裹，`[^"]` 的 `"` 没转义导致 bash 字符串提前关闭，后续 `node -e` 的 `(` 被当成 bash 语法
- **修复**：outputs 添加 `dev_version` / sed 改用单引号+变量拼接 `'...'"$VAR"'...'` / `node -e` 替换为 sed

### 问题 2：release.yml 缺少版本注入
- **根因**：推送 `v0.2.0` tag 时，`latest.json` 写入 `0.2.0`，但 `Cargo.toml` 仍为 `0.1.0`，安装后永远提示有更新
- **修复**：release.yml build job 新增 `Inject release version` 步骤，从 `${GITHUB_REF_NAME#v}` 提取版本号注入三处文件

### 问题 3：MSI 打包不支持 pre-release 版本标识符
- **错误**：`optional pre-release identifier in app version must be numeric-only and cannot be greater than 65535 for msi target`
- **根因**：MSI 要求版本号每段为纯数字且 ≤65535，而 `0.1.0-dev.abc1234` 含字母
- **修复**：区分两个版本号
  - `DEV_VERSION`（显示用，语义化）：`0.1.0-dev.abc1234` → 写入 `dev.json` + `package.json`
  - `BUILD_VERSION`（打包用，数字）：`0.1.0.<数字>` → 注入 `Cargo.toml` + `tauri.conf.json`
  - SHA 前4位十六进制 → 十进制（`$((16#${SHA_SHORT:0:4}))`，最大 65535）
- **前端适配**：`getLocalVersion()` 在 dev 构建时优先用 `__APP_VERSION__`（来自 package.json 的语义化版本），latest 构建时优先用 Tauri `getVersion()`

- `.github/workflows/release.yml` — 新增 Inject release version 步骤
