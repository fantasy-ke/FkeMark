## 1. P0：修复 Markdown 渲染和导出注入风险

### 问题
- 内置 Markdown 行内渲染器会把普通文本、链接地址、图片属性等内容直接拼接到 HTML 中。
- 恶意或异常 Markdown 可能通过标签、事件属性、危险协议或非法尺寸值进入预览和导出结果。
- HTML 导出维护了一套独立的简化 Markdown 转换逻辑，与应用预览渲染规则不一致。

### 修改
- 普通文本统一执行 HTML 转义，仅保留解析器主动生成的安全 HTML 片段。
- 链接和图片地址增加协议白名单，支持 http、https、mailto、锚点及相对路径，其他协议回退为安全地址。
- 图片的 src、alt、title、style 等属性统一执行属性转义。
- 图片宽高仅接受合法数字与 px、%、em、rem、vw、vh 等尺寸单位。
- HTML 导出复用 Markdown 预览渲染器，删除重复且安全边界不一致的转换实现。
- 更新核心测试，覆盖转义后的预期结果。

### 涉及文件
- `src/utils/markdown.ts`
- `src/utils/importExport.ts`
- `tests/core.test.ts`

## 2. P0：修复保存流程和更新安装安全边界

### 问题
- 回收站恢复和永久删除接口可接收任意路径，缺少“目标必须位于应用回收站目录内”的约束。
- 更新安装器文件名与安装路径缺少完整规范化和目录边界校验。
- 更新包在缺少 SHA-256 时仍可能继续处理，安装文件类型也缺少平台白名单。
- 顶栏保存按钮没有真正触发当前文档保存。
- 安装更新前未集中保存所有已修改文档，存在未保存内容丢失风险。

### 修改
- 对回收站路径执行 canonicalize，并限制恢复/永久删除目标必须位于应用回收站目录。
- 清理更新版本号和安装器文件名中的路径分隔符及非法值。
- 安装前校验安装器必须位于应用更新目录，并按平台限制可执行安装包扩展名。
- SHA-256 校验值改为必填，缺失时直接拒绝安装。
- 顶栏保存按钮接入实际保存函数。
- 安装更新前同步当前标签内容并逐一保存所有已修改且已有磁盘路径的文档。

### 涉及文件
- `src-tauri/src/file_system.rs`

## 4. 修复更新成功后误报“更新未生效”

### 问题
开发版构建使用 `VITE_APP_VERSION=dev-提交号` 作为真实显示版本，但 MSI/WiX 包版本仍必须保持纯数字。安装完成后，Rust 自愈逻辑读取应用包版本，与待安装记录中的 `dev-提交号` 比较，导致实际安装成功仍显示“更新未生效，已保留当前版本”。

### 修改
- 新增统一的当前构建版本读取逻辑，优先读取构建时注入的 `VITE_APP_VERSION`。
- 未注入显示版本的正式构建继续回退到 Tauri 应用包版本。
- 更新安装记录、安装完成校验和回滚记录统一使用同一版本来源。
- 保持开发版显示版本与安装后的校验版本一致，消除成功安装后的误报。

### 涉及文件
- `src-tauri/src/updater.rs`

- `cargo test`：2 个 Rust 测试通过。

## 9. 修复拖拽图片初始化显示失败

### 问题
图片拖拽上传成功且磁盘路径正确，但实时编辑和阅读模式显示图片加载失败；分栏模式可以正常显示，进入源码模式修改内容后图片也会恢复。原因是 Rust 上传完成事件先返回 Markdown 相对路径，编辑器直接用该相对路径替换上传占位节点。WebView 无法按文档目录解析该地址，而分栏预览及源码重新解析流程会额外执行本地资源路径转换，因此不同模式表现不一致。

### 修改
- 上传完成事件返回图片相对路径后，先通过 Tauri 资源路径转换生成 WebView 可加载的 URL。
- 转换完成后再将上传占位节点替换为正式图片节点。
- Markdown 序列化时仍保存 `./assets/图片名` 相对路径，保持文档可迁移性。
- 统一实时编辑、阅读和分栏模式首次上传后的图片显示结果。

### 涉及文件
- `src/components/Editor.tsx`
