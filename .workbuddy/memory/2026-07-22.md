## 1. P0：修复 Markdown 渲染和导出注入风险

### 问题
- 内置 Markdown 行内渲染器会把普通文本、链接地址、图片属性等内容直接拼接到 HTML 中。
- 恶意或异常 Markdown 可能通过标签、事件属性、危险协议或非法尺寸值进入预览和导出结果。
- HTML 导出维护了一套独立的简化 Markdown 转换逻辑，与应用预览渲染规则不一致。

### 修改
- 普通文本统一执行 HTML 转义，仅保留解析器主动生成的安全 HTML 片段。
- 链接和图片地址增加协议白名单，支持 http、https、mailto、锚点及相对路径，其他协议回退为安全地址。
- 图片的 src、alt、title、style 等属性统一执行属性转义。
- 图片宽高仅接受合法数字与 px、%、em、rem、vw、vh 等尺寸单位。
- HTML 导出复用 Markdown 预览渲染器，删除重复且安全边界不一致的转换实现。
- 更新核心测试，覆盖转义后的预期结果。

## 11. 兼容更新清单缺失 SHA-256 校验值

### 问题
软件检查到更新后执行应用内下载，会报错 `missing SHA-256 checksum; automatic installer download was rejected`，导致自动更新无法继续。

### 根因
2026 年 7 月 18 日发布的当前正式版 `latest.json` 生成时间早于更新清单内嵌 `sha256` 的流程调整，下载资源只包含文件名、URL 和大小。同一 GitHub Release 已包含 `SHA256SUMS`，但客户端未读取该文件回填哈希。后端安全校验按设计拒绝了缺少 SHA-256 的安装包。

### 修改
- 静态更新清单缺少 `sha256` 时，自动读取同一 Release 的 `SHA256SUMS`。
- 优先使用清单中的 `tagName` 锁定校验文件版本，避免 `latest` 切换期间读取到不同版本的哈希。
- GitHub API 降级路径优先使用 Release 资源中的 `SHA256SUMS` 下载地址，并保留按 tag 构造地址的兼容处理。
- 新增 sha256sum 标准格式解析，支持双空格、二进制 `*` 标记、带空格文件名和 `./` 相对路径。
- 只回填缺失的哈希，不覆盖新版清单已提供的 `sha256`。
- 下载前在前端再次检查校验值，仍缺失时显示中英文多语言错误提示。
- 保留 Rust 后端“缺少 SHA-256 即拒绝自动下载”的安全边界，未降低安装包完整性要求。
- 新增单元测试，覆盖校验清单格式和旧版更新清单的哈希回填流程。

## 12. 修复编辑弹窗边界与链接、网络图片交互

### 问题
- 链接和图片编辑弹窗虽然已禁止点击遮罩关闭，但靠近编辑区边缘时仍可能被裁切，只显示部分内容。
- 编辑器内临时菜单使用全局点击监听时，点击侧边栏、顶部栏或底部状态栏也会触发关闭，影响应用其他区域的正常操作。
- 实时编辑模式下已有链接不能稳定修改显示文本和地址，普通点击、双击及组合键点击也缺少明确区分。
- HTTP/HTTPS 网络图片地址需要明确保留为网络资源，不能被本地资源路径转换逻辑误处理。

### 根因
- 图片编辑浮窗按窗口坐标直接定位在编辑器容器内，没有把窗口坐标换算为编辑区坐标，也没有根据弹窗实际尺寸计算右侧和底部边界。
- 斜杠菜单、表格选择器、有序列表选择器、代码语言选择器及右键菜单各自维护定位或关闭逻辑，边界处理不一致，部分关闭监听挂载在整个文档上。
- 链接弹窗打开后依赖编辑器当前选区；输入框获取焦点后选区可能变化，保存时无法可靠定位原链接范围，也无法同时更新显示文本。
- 网络资源识别规则大小写敏感，且未统一排除 `blob:` 等无需本地路径转换的地址。

### 修改
- 新增统一浮层边界计算工具，根据弹窗真实宽高把位置限制在窗口或编辑区可视范围内。
- 链接弹窗、图片尺寸弹窗和图片地址编辑弹窗限制在编辑区域内显示，不再覆盖侧边栏、顶部栏和底部状态栏。
- 图片编辑弹窗将窗口点击坐标换算为编辑区本地坐标；靠近四角时自动向内调整，并为小窗口增加最大宽高和内部滚动。
- 为图片/表格右键菜单、斜杠菜单、表格网格选择器、有序列表选择器和代码语言选择器统一增加边界自适应。
- 将编辑器临时菜单的外部关闭监听收敛到编辑区域；点击侧边栏、顶部栏或底部状态栏不再误关闭，点击编辑视图空白区域仍可关闭非输入型菜单。
- 复查其他弹出组件：需要填写内容的链接、图片地址和图片尺寸弹窗只允许通过确定、取消或 Escape 显式关闭；非输入型临时菜单保留编辑视图内点击外部关闭。
- 打开已有链接时保存稳定的文档范围，保存时可同时修改显示文本和链接地址，并保留原文本上的其他格式标记。
- 实时编辑模式单击链接打开编辑弹窗；双击或 Ctrl/Command 点击链接时调用系统外部打开接口，交给默认浏览器处理，不在软件内部导航。
- 阅读模式单击链接继续使用系统默认浏览器打开，双击和 Ctrl/Command 点击同样立即外部打开。
- 外部链接统一校验 HTTP、HTTPS 和邮件协议；Tauri 调用系统应用失败时不再回退到 WebView，防止链接在软件内部打开。
- 本地资源识别改为大小写不敏感，明确保留 `http:`、`https:`、`data:` 和 `blob:` 地址；第三方 Markdown 引擎不再把这些网络或内存图片地址改写为本地资源协议。
- 新增浮层四角边界、两套 Markdown 引擎网络图片地址与外链协议校验回归测试。

### 涉及文件

## 16. 修复第三方 Markdown 引擎松散表格渲染

### 问题
从聊天或文档中复制的 Markdown 表格在每行之间带空行时，预览没有生成表格，而是显示为普通段落。

### 根因
第三方引擎使用 markdown-it，表格语法要求表头行、分隔行和内容行连续。空行会结束表格块；原管线没有在进入 markdown-it 前兼容清理表格内部空行，导致分隔行无法紧跟表头行。

### 修改
- 在第三方 Markdown 渲染入口增加松散表格归一化，只移除同一个表格块内部的空行。
- 保持代码块内内容不参与归一化，避免示例代码里的管道文本被误处理。
- 在归一化后再提取表格分隔线和列表元数据，保证 `data-separators` 仍能正确注入。
- 新增第三方引擎回归测试，覆盖表格行之间有空行时仍渲染为 `<table>`。

## 17. 修复引用渲染高度和空引用分裂

### 问题
引用语法在编辑区渲染后整体高度偏高；在部分内容中还会偶尔出现上下两个引用块，上方引用包含文字，下方引用为空，而分栏静态预览看起来正常或不明显。

### 根因
- 引用块自身上下 margin、padding 与内部段落默认 16px 底部间距叠加，导致单条引用显示偏高。
- 第三方引擎 markdown-it 遇到分隔后的单独空引用行（如 `> 文本` 后隔空行再出现 `>`）会生成第二个空 `blockquote`。
- 静态预览中的空 `blockquote` 没有段落内容，不太明显；进入 TipTap 编辑器后，TipTap 会按引用块 schema 自动补一个空段落，于是显示成下方独立的空引用块。

### 修改
- 收紧引用块上下间距、左侧缩进和左边框宽度，让引用展示更紧凑，更符合编辑器整体风格。
- 单独重置引用块内部段落间距，避免通用段落底部间距叠加到引用块中。
- 在第三方 Markdown 渲染后处理阶段移除空引用块，避免 TipTap 自动补空段落后形成可见的第二个空引用。
- 新增第三方引擎回归测试，覆盖分隔空引用不会生成第二个空引用块，以及仅包含空引用时回退为空段落。
