## 修复1：窗口最大化按钮无响应
- **根因**：`useTauriWindow().toggleMaximize()` → `plugin:window|toggle_maximize`，但 `src-tauri/capabilities/default.json` 仅授予 `allow-maximize`/`allow-unmaximize`，**缺少 `core:window:allow-toggle-maximize`**。被拒的 IPC 被 `safeTauriCall` 的 catch 静默吞掉 → 点击无反应。
- **按钮本身正常**：无 `disabled`、无 `pointer-events:none`（titlebar.css 已确认）。
- **修复**：
  - capabilities 补 `core:window:allow-toggle-maximize`
  - `TopBar.tsx` 新增 `isMaximized` prop，最大化时切换"还原"图标 + 同步 title/aria-label
  - `App.tsx` 传 `isMaximized={windowMaximized}`
  - i18n 补 `topbar.restore`（zh-CN/en）
- **commit**：`271e285`
- 注意：capabilities 改动需**重新构建 Rust 二进制**（`tauri build`/`tauri dev`）才生效，前端 HMR 无效。

## 修复2：分栏视图滚动同步
- **此前**：split 模式源码文本域与右预览为**独立滚动**（设计如此）。
- **修复**（`Editor.tsx`，commit `9cf76cd`）：
  - 新增 `previewScrollRef` 绑定右预览滚动容器
  - `handleSplitScroll`：以 `activeScrollRef` 记录"正在被用户滚动"的面板，避免 A→B 同步后 B 的 scroll 回灌成回路；**同一面板连续快速滚动不会被自身守卫挡住**，对面板回声被忽略
  - 配合 `requestAnimationFrame` 重置守卫 → 快速滚动仍流畅、不丢事件
  - 源码文本域与预览容器均挂 `onScroll={handleSplitScroll}`

## 修复3：四个功能迭代（commit `ec3e1ce` + `793307d`）

### ① 链接/图片单击直接编辑
- **之前**：链接仅 Typora 分隔符可见，URL 不可直接编辑；图片仅右键菜单调整尺寸
- **实现**：
  - 实时模式 `.editor-scroll` 容器添加 `onClickCapture`：
    - 单击 `<a.md-link>` → `editor.view.posAtDOM` 定位 → 弹 LinkDialog 预填 URL + 文本
    - 单击 `<img>` → 定位 ProseMirror node → 弹 ImageEditPopup（浮动内联弹窗，src + alt 输入框）
  - `applyLink()` 更新：检测选区是否在已有 link mark 内 → `unsetLink` + 重新 `setLink`（安全替换 href）
  - 新增 `applyImageEdit()`：通过 `tr.setNodeMarkup` 更新图片 src/alt
  - 新增 `.image-edit-popup` CSS（menus.css，与 link-dialog 风格一致）
- **注意**：阅读模式下不触发编辑（`isReadMode` 检查）

### ② 禁用分栏/源码视图右键菜单
- **修复**：源码 textarea、分栏 textarea、分栏预览容器均添加 `onContextMenu={(e) => e.preventDefault()}`
- **之前**：这三个区域无右键处理 → 浏览器默认菜单弹出

### ③ 修复实时/阅读模式图片加载失败
- **根因**：`asset.ts` 的 `isRelativeAssetPath()` 仅匹配 `./assets/` 和 `assets/` 前缀，其他相对路径（`./images/`、`../` 等）不会被 `convertFileSrc` 转换 → WebView 无法加载
- **修复**：改为匹配所有非绝对 URL 的相对路径（排除 `http:`/`https:`/`data:`/`#`/`/` 开头），使任意本地相对路径图片都能正确加载
- **说明**：分栏模式图片正常是因为 `dangerouslySetInnerHTML` 直接插入原始 HTML（src 已被 `markdownToPreviewHtml` 转换），而 live/read 模式经 TipTap `setContent` 后需从 ProseMirror node 重建 `<img>`——重建时的 src 来自 node.attrs，路径转换在 `markdownToHtml` 阶段完成，修复后两阶段一致

### ④ 所有视图模式支持文本检索
- **之前**：FindReplaceBar 仅基于 ProseMirror doc 搜索；源码模式被 `!isSourceMode` 隐藏；分栏模式 EditorContent 未渲染 → 搜索完全失效
- **实现**：
  - FindReplaceBar 新增 `forceTextMode`/`content`/`onContentChange` props
  - 文本模式分支：`findMatchesInText()` 在纯文本中正则搜索，`countLinesToIndex()` 计算行号滚动 textarea，`textarea.setSelectionRange()` 选中当前匹配
  - `doSearch` / `updateCurrentMatch` / `replaceCurrent` / `replaceAll` 全部双分支
  - 关闭查找栏时清除 textarea 选中
  - Editor.tsx：移除 `visible={findReplaceVisible && !isSourceMode}` → `visible={findReplaceVisible}`，源码/分栏模式传入 `forceTextMode={true}` + `content` + `onContentChange`
- **限制**：文本模式下 textarea 内无高亮（textarea 不支持富文本），但通过选中文本 + 计数提示给予视觉反馈
